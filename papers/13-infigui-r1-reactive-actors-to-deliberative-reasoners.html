TL;DR

Actor2Reasoner 两阶段训练。第一阶段 Reasoning Injection，用空间推理蒸馏把 teacher 的跨模态空间推理迁移给 MLLM；第二阶段 Deliberation Enhancement，用 RL 强化中间 sub-goal 和错误恢复场景。 摘要报告在 GUI grounding 与 trajectory tasks 上取得强表现；综述把它归为 reasoning-enhanced RL 架构，核心价值是把“先能操作”再“内化推理”的训练顺序系统化。

1. 要解决的问题

很多 GUI agent 是 reactive actor，只隐式推理，面对长任务、错误恢复和复杂空间关系时不够稳；手工 reasoning template 又不自适应。

换成 GUI agent 的语言，这个问题通常落在三件事上：界面状态部分可观测、正确反馈稀疏且延迟、静态示范无法覆盖真实软件变化。论文的价值就在于把这些问题中的一个或多个转成可训练的闭环信号。

2. 任务形式：输入、动作、奖励

观察屏幕截图、目标文本或自然语言指令，有时还包含裁剪区域和候选元素。
动作输出坐标、边界框或带坐标的 GUI action，例如 click(x,y)。
奖励坐标命中、IoU、距离衰减、高斯覆盖度或语义定位是否正确。

这个表是读 GUI Agent RL 论文最重要的入口：只要弄清楚模型看到了什么、能做什么、奖励从哪里来，就能判断它是算法贡献、环境贡献、奖励贡献还是系统贡献。

3. 方法与架构怎么跑

  1. InfiGUI-R1 的目标是把 reactive actor 变成 deliberative reasoner：模型不是直接点，而是先形成空间关系和子目标。
  2. Actor2Reasoner 第一阶段注入 reasoning：用 teacher 的跨模态空间推理教模型理解“左上角第二个按钮”“表格第三列”等关系。
  3. 第二阶段用 RL 强化 deliberation：在有错误恢复、子目标选择和复杂布局的任务里，用奖励让模型保留有用推理、抑制空泛推理。
  4. 它对应 GUI agent 的 System-2 路线：不是所有任务都需要长推理，但复杂界面需要可控的中间思考。

因此，这篇论文不是简单地“让大模型点屏幕”，而是围绕 observation-action-reward 契约重写训练闭环：先让动作可执行，再让反馈可验证，最后让策略能从成功和失败中更新。

4. 训练、奖励与数据流

  • 离线阶段先从静态轨迹、成功/失败对或规则可验证样本中学习，优点是便宜、安全、可复现。
  • 优化通常用 DPO、offline GRPO、Q-learning、IQL/CQL 或 value reranking，重点是避免策略偏离数据分布后无人纠正。
  • Grounding 训练必须把视觉空间误差转成可优化 reward；常见做法包括距离奖励、IoU、高斯奖励和裁剪重采样。
  • 如果使用 GRPO/RLVR，模型会在同一状态采样多个候选动作，用相对奖励更新策略，避免训练单独 critic。

读实验时要特别注意 reward 口径：有的论文评估单步坐标，有的评估整条任务成功，有的只报告系统吞吐或数据合成质量。不同口径不能直接相加或横向混算。

5. 关键设计与创新点

  • 把 GUI 任务改造成可验证奖励问题，让策略优化不必依赖昂贵的逐步人工标注。
  • 在不访问真实环境的前提下利用轨迹、偏好或 value signal，降低安全风险和交互成本。
  • 把视觉定位从硬命中扩展到连续或主动感知反馈，使坐标错误能产生可学习梯度。

这些创新点的共同目标，是把 GUI 交互从一次性 prompt 调用变成可迭代优化的训练系统。

6. 实验怎么读

摘要报告在 GUI grounding 与 trajectory tasks 上取得强表现；综述把它归为 reasoning-enhanced RL 架构，核心价值是把“先能操作”再“内化推理”的训练顺序系统化。

这类实验通常需要同时看三组数字：第一是任务成功率或 grounding accuracy；第二是与 SFT、BC、prompting、GPT-4V/闭源模型或开源基线的对比；第三是环境成本，例如 rollout 速度、样本数量、标注成本和并行规模。

7. 复现或落地时要准备什么

  1. 先固定 action schema，并写一个能把模型输出解析成真实 GUI 操作的 adapter。
  2. 准备 verifier：至少要能判断终局成功；如果是 grounding 论文，还要能计算坐标/区域奖励。
  3. 离线训练要检查轨迹覆盖和负样本质量，避免模型学到只在数据集状态里有效的捷径。
  4. Grounding 实验要保留原始分辨率、元素框和坐标归一化规则，否则不同 benchmark 不可比。

如果只能先复现一小部分，建议优先复现 action schema、verifier 和一组可重置任务；没有这三件事，后续 RL 指标很难可信。

8. 局限与读法

  • 单步 grounding 分数高不等于多轮任务成功；坐标准但点错流程仍会失败。
  • 连续奖励或裁剪策略可能对特定 benchmark 有利，跨分辨率、跨应用仍需验证。
  • 离线轨迹覆盖不到的状态仍是风险；模型一旦偏离专家路径，可能缺少纠正样本。

它位于“把视觉定位做准”的支线：没有可靠 grounding，多轮策略再强也会在第一步点击或输入上失败。
